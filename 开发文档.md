## 二、个人信息配置

## 三、如何写博客

### 1. 创建新文章

**方法一：命令行创建**
```bash
cd hexo-site
hexo new "文章标题"
```

**方法二：手动创建文件**

在 `source/_posts/` 目录下创建文件，命名格式：`YYYY-MM-DD-标题.md`

### 2. 文章模板（Front-matter）

```yaml
---
title: 文章标题                    # 必填
date: 2025-03-18 10:00:00        # 发布日期
updated: 2025-03-18 10:00:00     # 更新日期
categories:                      # 分类（只能一个）
  - 技术教程
tags:                            # 标签（可多个）
  - Python
  - 编程
toc: true                        # 是否显示目录
excerpt: 文章摘要，会显示在列表中
cover: /images/cover.jpg         # 封面图片（可选）
---

# 正文开始

这里是文章内容，支持 **Markdown** 语法。
```

### 3. Markdown 常用语法

```markdown
# 一级标题
## 二级标题
### 三级标题

**粗体** *斜体* ~~删除线~~

- 无序列表项 1
- 无序列表项 2

1. 有序列表项 1
2. 有序列表项 2

[链接文字](https://example.com)

![图片描述](/images/photo.jpg)

`行内代码`

​```python
# 代码块
def hello():
    print("Hello, World!")
​```

> 引用文字

---

表格：
| 列1 | 列2 |
|-----|-----|
| 内容 | 内容 |
```

### 4. 数学公式（MathJax）

```markdown
行内公式：$E = mc^2$

块级公式：
$$
\int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi}
$$
```

### 5. Mermaid 图表

```markdown

​```

## 六、常用命令

```bash
# 进入项目目录
cd hexo-site

# 创建新文章
hexo new "文章标题"

# 本地预览（http://localhost:4000）
hexo server

# 生成静态文件
hexo generate

# 清理缓存
hexo clean

# 清理后重新生成并预览
hexo clean && hexo server
```

## 七、部署到 GitHub Pages

### 1. 安装部署插件

```bash
cd hexo-site
npm install hexo-deployer-git --save
```

### 2. 配置 `_config.yml`

```yaml
deploy:
  type: git
  repo: https://github.com/CoolPig0720/CoolPig0720.github.io.git
  branch: main
  message: "Site updated: {{ now('YYYY-MM-DD HH:mm:ss') }}"
```

### 3. 部署命令

```bash
hexo clean
hexo generate
hexo deploy
```

或者简写：
```bash
hexo clean && hexo g -d
```

## 九、注意事项

1. **文件编码**：所有 `.md` 文件使用 **UTF-8** 编码
2. **YAML 格式**：配置文件注意**英文冒号**后要有空格
3. **日期格式**：`YYYY-MM-DD HH:mm:ss`
4. **图片路径**：使用 `/images/xxx.jpg` 格式（绝对路径）
5. **修改后**：运行 `hexo clean && hexo server` 查看效果
6. **部署前**：先本地预览确认无误再部署

## 十、常见问题

### Q: 修改后页面没有变化？

```bash
hexo clean
hexo server
```

### Q: 图片无法显示？

检查路径是否正确，使用绝对路径 `/images/xxx.jpg`

### Q: YAML 解析错误？

检查冒号后是否有空格，中文冒号改为英文冒号

### Q: 如何添加新页面？

1. 在 `source/` 下创建新目录
2. 创建 `index.md` 文件
3. 在 `_config.butterfly.yml` 的 `menu` 中添加导航
