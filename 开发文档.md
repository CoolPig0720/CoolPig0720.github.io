## 三、自定义样式系统

### 1. inject CSS/JS 机制

所有自定义样式通过 `_config.butterfly.yml` 的 `inject.bottom` 注入，无需修改主题源码。

**当前注入内容：**

| 功能 | 说明 |
|------|------|
| 网站标题放大 | `#blog-info .site-name` 字号1.4em |
| 图标文字对齐 | `vertical-align: middle` |
| 导航标签样式 | 悬浮效果+搜索按钮圆角 |
| 侧边栏样式 | 隐藏标签统计、放大统计数字 |
| 文章排版优化 | 正文字号、行高、间距调整 |
| 表格居中 | 文章内表格文字居中 |
| 归档页分类分组 | 分类卡片+可展开交互 |
| 三栏布局 | 左侧栏+主内容+右侧TOC |
| 最新文章标题截断 | 单行显示+省略号 |
| TOC右置 | 桌面端TOC移至右侧独立容器 |

### 2. 主题补丁系统 (`themes/_patches/`)

由于 `node_modules/hexo-theme-butterfly` 中的文件会被 `npm install` 覆盖，所有对主题源码的修改必须放在 `themes/_patches/` 目录中，CI构建时自动覆盖。

**当前补丁文件：**

| 补丁文件 | 覆盖目标 | 功能 |
|----------|----------|------|
| `layout/archive.pug` | 归档页模板 | 按分类分组显示文章 |
| `layout/includes/widget/card_post_toc.pug` | TOC组件 | TOC层级显示优化 |
| `source/js/main.js` | 主JS | 归档页展开/收起交互 |
| `source/js/utils.js` | 工具JS | 辅助功能 |

**添加新补丁时，需同步修改：**
1. 在 `themes/_patches/` 中放入补丁文件
2. 在 `.github/workflows/deploy.yml` 的 `Apply custom theme patches` 步骤中添加 `cp` 命令

## 四、页面定制指南

### 首页 (`source/index.md`)

- 使用 HTML + CSS 自定义样式
- 包含欢迎区域、关于本站、联系方式、底部引言
- **已适配暗色模式**：所有颜色使用 CSS 变量（`var(--card-bg)`, `var(--font-color)` 等）
- 修改内容：搜索注释标记 `【修改】` 即可定位

### 关于我页面 (`source/cv/index.md`)

- 使用 HTML + CSS 实现紧凑简历布局
- 基准字号 `1.20em`（通过 `.cv-page` 容器控制，所有子元素自动缩放）
- **已适配暗色模式**：使用 CSS 变量
- **隐私保护**：手机号部分隐藏，微信号改为邮件联系

**布局结构：**

| 模块 | 实现方式 |
|------|----------|
| 基本信息 | 表格（2列×3行），标签列用 `em` 宽度+`nowrap` |
| 教育经历 | 时间线样式（`::before` 伪元素生成圆点+竖线） |
| 奖项 | 行内标签（`flex-wrap` + 浅金色背景） |
| 工作经历 | 时间线 + h4子标题（左边框）+ 自定义bullet段落 |
| 项目经历 | flex并排卡片（`.cv-projects` 容器，移动端自动竖排） |
| 技术能力 | 行内代码标签样式（`code` 元素 + 圆角背景） |
| 游戏经历 | 双列网格（`.cv-game` inline-block） |

**⚠️ 渲染兼容性注意：** `hexo-renderer-marked` v6 中，`<div>` 内部**禁用** `<ul>`/`<li>`/`<b>` 元素（会被转义为纯文本）。请使用 `<p class="cv-bullet">` 替代列表，`<strong>` 替代 `<b>`。

### 归档页

- 已改造为按分类分组展示
- 支持点击分类标题展开/收起
- 补丁文件：`themes/_patches/layout/archive.pug`

### 其他页面

论文、报告、教学、作品集页面模板已创建，导航栏中已注释（按需启用）。

## 五、如何写博客

### 1. 创建新文章

**方法一：命令行创建**
```bash
cd hexo-site
hexo new "文章标题"
```

**方法二：手动创建文件**

在 `source/_posts/` 目录下创建文件，命名格式：`YYYY-MM-DD-标题.md`

### 2. 文章模板（Front-matter）

```yaml
---
title: 文章标题                    # 必填
date: 2025-03-18 10:00:00        # 发布日期
categories:                      # 分类（只能一个）
  - 技术教程
tags:                            # 标签（可多个）
  - Python
  - 编程
toc: true                        # 是否显示目录
---
```

### 3. 面试类文章规范

面试类博客**仅归属**"面试问题"分类，不添加其他分类：

```yaml
---
title: 面试题标题
categories:
  - 面试问题
---
```

## 七、常用命令

```bash
# 进入项目目录
cd hexo-site

# 本地预览（http://localhost:4000）
hexo server

# 清理缓存后预览
hexo clean; hexo server

# 生成静态文件
hexo generate

# 清理后重新生成
hexo clean; hexo generate
```

> **注意**：PowerShell 中使用分号 `;` 分隔命令，不用 `&&`。

## 八、部署

### 自动部署（推荐）

代码推送到 `master` 或 `main` 分支后，GitHub Actions 自动构建部署。

**部署流程：**
1. `npm install` 安装依赖
2. 复制 `themes/_patches/` 中的补丁到 `node_modules/hexo-theme-butterfly/`
3. `hexo generate` 生成静态文件
4. 部署 `hexo-site/public` 到 `gh-pages` 分支

> **⚠️ 重要**：本地修改不会直接生效，必须推送到 GitHub 远程仓库后由 CI 构建。

### 手动部署

```bash
cd hexo-site
hexo clean; hexo generate; hexo deploy
```

## 九、暗色模式适配规范

自定义页面中必须使用 CSS 变量替代硬编码颜色：

| 用途 | 硬编码（❌） | CSS变量（✅） |
|------|------------|-------------|
| 卡片背景 | `white` / `#fff` | `var(--card-bg)` |
| 正文颜色 | `#2c3e50` / `#333` | `var(--font-color)` |
| 高亮色 | `#3498db` | `var(--text-highlight-color, #667eea)` |
| 辅助文字 | `#7f8c8d` / `#95a5a6` | `var(--font-color)` + `opacity` |
| 边框色 | `#ddd` | `var(--dark-border, #ddd)` |

## 十、注意事项

1. **文件编码**：所有 `.md` 文件使用 **UTF-8** 编码
2. **YAML 格式**：配置文件注意**英文冒号**后要有空格
3. **主题修改**：勿直接修改 `node_modules/` 下的文件，改用 `themes/_patches/` 或 `inject`
4. **HTML in Markdown**：`<div>` 内禁用 `<ul>`/`<li>`/`<b>`，用 `<p>`+`<strong>` 替代
5. **尺寸单位**：优先使用 `em` 而非 `px`，确保字体缩放时布局不错乱
6. **部署生效**：本地修改需推送到 GitHub 才能在线上生效
